# v0.3 roadmap

Captured items deferred from v0.2 and ideas surfaced post-M10.
Nothing here is committed to a milestone yet; this is a parking lot.

## Bigger features

### Rule-vs-fact disambiguation in consolidation

**Problem.** Users sometimes type durable project rules into the
agent - "don't forget to never add a new function without a unit
test", "we always run lint before merging", "memory mustn't store
PII". These end up as ordinary observations in engram. But they'd
be more useful as instructions in the project's `CLAUDE.md` /
`AGENTS.md`, because:

- Rules need to fire on every relevant action; engram queries
  fire when the agent thinks to call them.
- Project-instruction files are checked into version control and
  visible to the whole team.

**Sketch.** Extend the consolidator's structured-output schema to
classify each surfaced observation into `{decision, fact, rule,
gotcha}`. Rule-tagged items land in a separate page
`wiki/_rules/uncommitted.md` AND surface a lint finding:

> "This rule isn't in your project's CLAUDE.md — consider adding it
> there so the agent sees it on every turn, not just when it
> searches memory."

Detection heuristics: starts with `always`, `never`, `don't`,
`remember to`, `before X always`, `as a rule`, or contains
`(rule|policy|convention)`. Easy, ~50 lines plus the lint surface.

The output is a *suggestion only* - engram never auto-edits a
user's CLAUDE.md / AGENTS.md / project instructions file.

### Bootstrap an existing project's history into the wiki

**Problem.** Today engram starts empty for any project. For
existing projects with months of decisions in `git log`, `docs/`,
`README.md`, code comments - that's a lot of warm context the user
already paid for once, and we throw it away.

**Sketch.** A `engram bootstrap` one-shot subcommand. Inputs:

- `git log --oneline -p` filtered to commits with substantive
  messages (e.g. > 100 chars body, conventional-commit prefix, or
  matching `^(feat|fix|refactor|design):`).
- README + `docs/*.md` + `CLAUDE.md` if present.
- Module-level doc-comments from the codebase (`find . -name
  "*.rs" | xargs head -30`, language-aware).

LLM consolidates these into:

- `wiki/concepts/<topic>.md` per recurring theme
- `wiki/decisions/0001-…md` per ADR-shaped commit
- A bootstrap handoff page summarising "what I learned about this
  project"

Bootstrap pages get `tier: bootstrap` in their frontmatter so the
lint pass treats them as lower-confidence than session-grown pages,
and they participate normally in the decay sweep.

**Risk.** LLM-fabricated detail polluting the wiki. Mitigations:
the user reviews + commits the wiki tree before keeping (it's a git
repo); bootstrap pages are explicitly tagged so a `forget-sweep` can
target them; the LLM is instructed to mark uncertain claims with a
`(needs verification)` suffix.

### CI matrix tightening

Currently `ci.yml` runs fmt + clippy + test + cargo-deny + cargo-
audit + gitleaks on `ubuntu-latest` only. Worth adding:

- macOS runner (catches stat / SELinux-label assumptions)
- A `cargo build --release` step so we don't ship code that fails
  release-mode optimisation
- A docker-build smoke step that confirms `docker run --rm
  $IMAGE --version` works after every push

## Smaller polish

### Hook scripts as a generated artefact

The 21 hook scripts live in `hooks/*/*.sh` as files. There's no
regenerator. The shape is identical across agents (only
`agent=open-code` vs `agent=codex` etc. differs). A `bin/regen-hooks`
script that templates them from `CLAUDE_CODE_EVENTS` + a per-agent
name would let us add a new event in one place.

### Consolidator prompt templating

Both `build_batch_request` and `build_request` in
`crates/engram-consolidate/src/consolidator.rs` `push_str` their
way to a prompt. Constants are fine for two prompts. If the count
grows to four+ a small templating helper would earn its keep.

### `Handoff` field grouping

The struct has 16 public fields. The audit suggested grouping into
nested sub-structs. Not urgent; would force accessor churn across
the codebase for no operational win. Revisit if/when adding a 17th
field.

### `cargo install engram` + `mise use -g github:semantic-craft/engram`

Today the only install paths are docker + `git clone + cargo
build`. ai-jail ships a release-binary GitHub Actions workflow that
publishes to crates.io + GitHub Releases (tar.gz + sha256 per OS
target). Mirror that pattern for users who want a host-side binary
without the rust toolchain.
